# Домашнее задание №2 - КПО
Разработать модуль учета финансов для приложения банка.

## Техническая реализация
Для реализации всех задач была сделана разбивка проекта на несколько частей. В отдельные части вынесена UI библиотека, библиотека менеджера окон. Также как отдельные 2 части реализованы сервисы банка и клиентская (визуальная часть приложения).
### Функционал
Реализован весь функционал, который требовался по задаче. В том числе сделан экспорт/импорт необходимых видов файлов (внимание: импорт некоторых файлов может отклоняться по причине того, что файлы имеют некорректный вид. Это может значит, например, что в файле в один момент времени у был отрицательный баланс у счета - загрузка таких файлов невозможна, т. к. сами они очевидно не имеют смысла). Все данные валидируются, клиентская часть делает запросы серверной в строго регламентированных местах. Получение данных делается через фасады.

### Применение SOLID
Принципы SOLID применяются в приложении везде:
*  SRP - применяется во всех классах приложения. Каждый класс имеет единое назначение. Наиболее наглядно видно на примерах сервисов, например, сервиса ViewManager, а также на примере фасадов (BankAccountFacade, CategoryFacade и. т. д.)
* OCP - также соблюдается. Инкапсуляция работает отлично. Изменение данных ограничено максимально сильно, чтобы не было ненужного доступа извне. В частности, применено для доменных объектов (BankAccount, Operation, Category)
* LSP - буквально по определению применяется во всех сервисах, а также в окнах приложения 
* ISP - интерфейсы максимально сегрегированы чтобы они выполняли только одну функцию за раз. Хорошо видно на примере IViewGetsCommandTime
* DIP - все сервисы работают для того, чтобы выполнять этот принцип. Также используется в UI-библиотеке

### Применение паттернов
Программа построена на паттернах. Вот примененные:
* Фабрика - используются для создания всех доменных объектов с различными входными данными. Нужно для корректной валидации объектов (BankAccountFabric, BankAccountRetrieveFabric, BankAccountDuplicateFabric и аналогичные для Category и Operation)
* Наблюдатель - используется для наблюдения за изменением контейнера категорий - при удалении категории у всех операций должно обновиться поле категории на пустое (CategoryUpdateObserver)
* Стратегия - используется для реализации механизма экспорта в программе (IExportStrategy). Используется для реализации механизма калькулятора баланса (BalanceUpdaterContext и BalanceCalculationStrategy) 
* Фасад - используется в клиентской части для преобразования данных серверной части в отформатированный с помощью разных сервисов текст (FormatterFacade). Также используется в серверной части для обработки запросов создания, изменения, удаления доменных объектов. Помимо этого используется как инструмент запросов клиентской части для подсчета статистики
* Декоратор - используется для "навешивания" на менеджер команд дополнительных оберток (в данном случае считается время выполнения команд) (CommandTimeDecorator). Также используется для финальной валидации контейнера операторов перед добавлением операции в контейнер (OperationContainerValidationDecorator)
* Команда - используется для передачи запросов на изменение данных серверной части от клиентской к серверной части приложения (IObjectCommand и реализации)
* Посредник - используется для передачи данных от декоратора команд к менеджеру окон (CommandTimeMediator)
* Шаблонный метод - используется для импорта данных (DataImporter и реализации). Как сервис, обрабатывающий использование шаблонного метода используется ImportContext

### Иное
Для транспортировки данных используются DataTransferObjects. В частности, используются DTO для транспортировки данных доменных объектов, чтобы доменные объекты не перемещались внутри программы. Это часть защиты данных внутри программы.

### DI-контейнер
Применен DI-контейнер для всех необходимых зависимостей. В частности, использована библиотека Scrutor для применения декораторов в DI-контейнере.

### Библиотеки
Применена стандартная библиотека для DI-контейнера, библиотека Scrutor для декораторов, библиотека YamlDotNet для импорта данных из формата Yaml.
